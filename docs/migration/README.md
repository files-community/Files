# Migration

These docs describe how to move from the current implementation to the target
architecture. Migration docs should name the current code, the target design,
the staged steps, and the verification needed at each step.

## Suggested migration-doc shape

- Current implementation
- Target architecture
- Migration stages
- Compatibility and rollout notes
- Verification checklist
- Cleanup criteria
